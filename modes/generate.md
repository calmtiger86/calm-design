# Mode A: Generate — 제로에서 디자인 생성

calm-design의 4가지 모드 중 가장 기본. 사용자 의도만 받아서 9-섹션 DESIGN.md → 코드 → 셀프-크리틱까지 한 번에 수행.

## 트리거 키워드

다음 키워드 중 하나라도 등장하면 Mode A로 진입:
- "디자인 만들어줘", "디자인 짜줘", "UI 짜줘", "화면 만들어줘"
- "랜딩페이지", "대시보드", "마이페이지", "로그인 화면" 등 페이지 종류 + 동사 명령
- "design X", "make a Y page", "build me a Z screen"

> 단, "토스 스타일", "Linear 처럼" 등 레퍼런스 키워드가 동시에 등장하면 → Mode C(Match-Reference) 우선
>
> "3가지 안", "여러 옵션" 등이 동시 등장하면 → Mode D(Multi-Variant) 우선

## 진입 직후 9단계 워크플로우

### Step 1. 사용자 입력 분석 (의도 분류기)

사용자 입력 텍스트에 대해 **5축 분류**를 동시 실행. 모든 축은 신뢰도 점수(0-100)와 함께 산출.

#### 축 1: 페이지 타입 (필수)

| 키워드 시그널 | 분류 | output-engines 매핑 |
|---|---|---|
| "랜딩페이지", "랜딩", "마케팅 사이트", "홈페이지" | landing | `html-tailwind.md` Section 3a |
| "대시보드", "관리자", "어드민", "콘솔", "워크스페이스" | dashboard | `html-tailwind.md` Section 3b 또는 `react-shadcn.md` |
| "로그인", "회원가입", "인증", "비밀번호 재설정", "OTP" | auth | `html-tailwind.md` Section 3c |
| "모바일 앱", "앱 화면", "iOS", "Android", "릴스 스타일" | mobile | `html-tailwind.md` Section 3d |
| "블로그", "아티클", "포스트", "에디토리얼", "콘텐츠" | content | `html-tailwind.md` Section 3e |
| "단일 컴포넌트", "버튼만", "카드만", "폼 하나" | component | 단일 출력 (페이지 골격 X) |

신뢰도 < 60이면 → AskUserQuestion 트리거 ("어떤 종류의 화면인가요?")

#### 축 2: 브랜드 톤 (필수)

`prompt-enhancement.md` Section B.1 한국어 분위기 단어 매핑 적용. 사용자가 톤 단어 미언급 시 기본값 = "차분 + 신뢰" (calm-design 기본 정체성).

#### 축 3: 명시적 다이얼 값 (선택)

```
사용자가 "VARIANCE=8" 또는 "강렬한 모션 9" 명시 → 그대로 사용
미명시 → 톤·페이지 타입에서 자동 추론
```

다이얼 자동 추론 매핑:

| 페이지 타입 + 톤 | VARIANCE | MOTION | DENSITY |
|---|:-:|:-:|:-:|
| landing + 차분 | 5 | 4 | 4 |
| landing + 트렌디 | 8 | 7 | 4 |
| landing + 강렬 | 8 | 8 | 5 |
| dashboard + 차분 | 4 | 4 | 7 |
| dashboard + 데이터 중심 | 4 | 3 | 8 |
| auth | 3 | 3 | 3 |
| mobile + 차분 | 5 | 5 | 5 |
| content/blog | 3 | 3 | 4 |

#### 축 4: 출력 형식 (선택)

| 키워드 시그널 | 출력 엔진 |
|---|---|
| "HTML로", "한 파일로", "바로 보고 싶어", 시그널 부재 | `html-tailwind` (기본) |
| "React로", "Next.js로", "TSX 파일", "컴포넌트 라이브러리" | `react-shadcn` |
| "DESIGN.md만 줘", "디자인 시스템 문서만" | DESIGN.md 단독 (Step 6-7 스킵) |
| "preview 카탈로그", "디자인 시스템 시각화" | `preview-catalog` (Phase 1+) |
| "Figma 명세" | `figma-export` (Phase 1+) |

#### 축 5: 다른 모드 트리거 시그널 (Mode A 종료 신호)

| 키워드 발견 | 전환 모드 | 우선순위 |
|---|---|:-:|
| "토스 스타일", "Linear처럼", "Vercel 풍" 등 브랜드 + "처럼/스타일" | Mode C (Match-Reference) | 2 |
| "3가지 안", "여러 옵션", "다양하게", "변형" | Mode D (Multi-Variant) | 1 |
| "이거 다듬어줘", "업그레이드", "개선" + 기존 코드 첨부 | Mode B (Upgrade) | 3 |

여러 시그널 동시 발견 시 우선순위 D > C > B 순. Mode A 진행 종료, 해당 모드의 `modes/{mode}.md` 로드 후 재시작.

#### Step 1 출력 (구조화 객체)

```yaml
intent_classification:
  page_type: dashboard       # confidence: 92
  brand_tone: 차분 + 신뢰     # confidence: 78
  explicit_dials: null
  output_engine: html-tailwind  # confidence: 60 (기본값)
  mode_switch: null            # null이면 Mode A 계속 진행

inferred_dials:
  variance: 4
  motion: 4
  density: 7
  language: ko
```

이 객체가 Step 2로 전달.

명확하지 않거나 신뢰도 60 미만 항목 1개 이상 → **AskUserQuestion** 트리거 (`prompt-enhancement.md` Section F의 실패 케이스 참조).

### Step 2. 4-다이얼 자동 추론

`SKILL.md` Section 2의 자연어 매핑 표 적용:

```
사용자 입력 → 다이얼 값
- "차분한 SaaS 대시보드" → VARIANCE=4, MOTION=4, DENSITY=7, LANGUAGE=ko
- "역동적인 랜딩페이지" → VARIANCE=8, MOTION=8, DENSITY=4, LANGUAGE=auto
- 명시값 있으면 그것 우선
```

### Step 3. 페이지 구조 자동 생성

`prompt-enhancement.md` Section C의 페이지 타입별 표준 섹션 적용. 예:

```
입력: "한국 B2B SaaS 대시보드"
출력 구조:
1. Sidebar (Logo + Nav 6개 + 사용자 프로필 하단)
2. Top Bar (검색 ⌘K + 알림 + 사용자 메뉴)
3. Page Header (제목 + Primary Action)
4. KPI Cards (Bento Grid 4개, 비대칭)
5. Main Chart (라인 또는 바, 풀-너비)
6. Secondary: Recent Activity Feed
```

### Step 4. 분위기 증폭

`prompt-enhancement.md` Section B 적용:

```
"차분한" → ["Stone neutrals", "MOTION_INTENSITY=4 미만", "longer transitions 400-600ms",
            "single muted accent (Emerald or Slate)", "generous line-height"]
```

### Step 5. 9-섹션 DESIGN.md 생성

`references/design-md-spec.md`의 정확한 포맷 준수. **이 단계가 1차 산출물이며 절대 건너뛰지 않는다.** 코드부터 출력하면 안 됨.

생성된 DESIGN.md는 `.calm-design/DESIGN.md`로 저장.

#### DESIGN.md 생성 시 체크
- 9섹션 모두 비어있지 않음 (각 섹션 ≥ 50자)
- Section 9 (Anti-Patterns) — 프로젝트 고유 금지만, 글로벌 금지(`ai-tells-blocklist.md`)와 중복 금지
- Section 1에 다이얼 값 + 모드 + 언어 헤더 명시:
  ```
  > Generated by calm-design · LANGUAGE: ko · VARIANCE: 4 · MOTION: 4 · DENSITY: 7 · Mode: A
  ```

### Step 6. 출력 엔진 결정

`SKILL.md` Section 4의 표 적용:

```
사용자 시그널 부재 → 기본 HTML 모드 (output-engines/html-tailwind.md 로드)
"React로", "Next.js" → React 모드 (output-engines/react-shadcn.md 로드)
"DESIGN.md만 줘" → Step 6 스킵, Step 7으로 직행
```

### Step 7. 코드 변환 (출력 엔진 적용)

DESIGN.md → 선택한 출력 엔진의 템플릿으로 변환. 라이브러리 정책 자동 로드:
- HTML 모드: `library-policies/lucide.md`, `library-policies/pretendard.md`
- React 모드: 위 + `library-policies/shadcn-ui.md`, `framer-motion.md`

**ALWAYS 로드되는 5개 references는 이미 코드 생성 결정에 반영돼 있음** (Inter 금지, Pure Black 금지, `min-h-[100dvh]` 강제 등).

### Step 8. 셀프-크리틱 루프 자동 트리거

사용자가 `--no-critique`를 명시하지 않은 한 자동 실행. `references/self-critique-loop.md`의 5단계:

```
[1] Generate (이미 끝남)
[2] Render+Capture (Cowork: visualize MCP / Claude Code: Playwright / fallback: 정적 분석만)
[3] Vision Self-Analysis (Phase 0은 코드 텍스트 추론 위주)
[4] Pre-Flight 21항목 채점
[5] 핀포인트 재생성 (실패 항목만, 최대 3회)
```

### Step 9. 사용자에게 4가지 산출물 전달

```markdown
## ✅ calm-design Mode A 완료

[ DESIGN.md ] (.calm-design/DESIGN.md)
[ 코드 출력 ] (HTML 또는 React 파일들)
[ Pre-Flight Report ] (21항목 ✅/⚠️/❌ + 재생성 횟수)
[ 다이얼 한 줄 ] LANGUAGE=ko · VARIANCE=4 · MOTION=4 · DENSITY=7 · Mode A
```

## 실패 처리

각 단계 실패 시:

| 단계 | 실패 시나리오 | 처리 |
|---|---|---|
| Step 1-2 | 의도 모호, 모순 다이얼 | AskUserQuestion 트리거 |
| Step 5 | DESIGN.md 9섹션 검증 실패 | 자동 재생성 (최대 3회), 그래도 실패 시 부분 결과 + 빈 섹션 표시 |
| Step 7 | 코드 변환 오류 (라이브러리 미지원 등) | 사용자에게 출력 엔진 변경 제안 |
| Step 8 | 셀프-크리틱 3회 시도 후 ❌ 잔존 | 부분 결과 + 미통과 리포트 동봉 |

## Mode A 사용 예시

### 예시 1: 한국 SaaS 대시보드

```
사용자: "한국 B2B SaaS 대시보드 디자인 만들어줘. 차분하고 데이터 많은 톤으로."

→ Step 1: 페이지=대시보드, 톤=차분, LANGUAGE=ko, 출력=기본(HTML)
→ Step 2: VARIANCE=4, MOTION=4, DENSITY=7, LANGUAGE=ko
→ Step 3: Sidebar/TopBar/PageHeader/KPICards/MainChart/ActivityFeed 6 섹션
→ Step 4: ["Stone neutrals", "tabular numerics", "structured grid", ...]
→ Step 5: DESIGN.md 9섹션 생성
   - Section 1: "차분한 한국 B2B SaaS 톤. Stone-50 캔버스에 단일 Emerald 액센트, 데이터 강조하되 화려하지 않은 신뢰감."
   - Section 2: Canvas/Surface/Ink/Mute/Border/Accent 6컬러
   - Section 3: Pretendard, word-break:keep-all, leading-relaxed
   - ... (9섹션 완성)
→ Step 6: HTML 모드 자동 선택
→ Step 7: HTML+Tailwind CDN+Pretendard+lucide+Motion One 템플릿 변환
→ Step 8: 셀프-크리틱 21항목 → 19✅ 1⚠️ 1❌ → 핀포인트 재생성 → 21✅ 통과
→ Step 9: 4가지 산출물 전달
```

### 예시 2: 마케팅 랜딩페이지

```
사용자: "AI 영상 편집 SaaS 랜딩. 트렌디하고 한국어로."

→ Step 1: 페이지=랜딩, 톤=트렌디, LANGUAGE=ko
→ Step 2: VARIANCE=7, MOTION=7, DENSITY=4
→ Step 3: 8 섹션 (Nav/Hero/Social Proof/Features/Use Cases/Pricing/CTA Banner/Footer)
→ Step 4: ["Bento Grid", "inline image typography", "clamp() responsive", "Cabinet Grotesk display"]
→ Step 5-9 동일 흐름
```

## 다른 모드와의 분기 (재확인)

Mode A 진입 후에도 사용자 입력을 다시 보고 다음 시그널 발견 시 다른 모드로 자동 전환:
- "토스처럼" → Mode C로 전환 (Match-Reference)
- "3가지 안" → Mode D로 전환 (Multi-Variant)
- "이거 다듬어줘" + 기존 코드 첨부 → Mode B로 전환 (Upgrade)

이 전환은 Step 2 직전에 마지막 검증.

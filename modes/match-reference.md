# Mode C: Match-Reference — 레퍼런스 자동 매칭

calm-design의 4가지 모드 중 세 번째. 사용자가 "토스 스타일", "Linear 처럼" 같은 레퍼런스 키워드로 요청하면 `reference-library/`에서 적합 브랜드 3개 후보를 자동 추천하고 적용 코드 생성.

> Mode A(Generate)는 제로에서 / Mode B(Upgrade)는 기존 개선 / **Mode C는 특정 레퍼런스에 정렬**.

## 트리거 키워드

다음 패턴 발견 시 Mode C 진입:
- "{브랜드명}처럼", "{브랜드명} 스타일로", "{브랜드명} 풍"
  - 한국: "토스처럼", "당근같이", "카카오 톤"
  - 글로벌: "Linear 스타일", "Vercel 같이", "Stripe-style"
- "이 레퍼런스 따라해줘", "참고할 디자인", "moodboard 매칭"
- 카테고리 시그널: "한국 핀테크 톤", "B2B SaaS 차분한", "이커머스 친근한"

> 다른 모드 키워드 충돌 시 우선순위: D > **C** > B > A

## 9단계 워크플로우

### Step 1. 레퍼런스 의도 추출

사용자 입력에서 다음 추출:
1. **명시 브랜드**: "토스처럼" → slug=`toss`
2. **카테고리**: "한국 핀테크" → category=`fintech` ∩ `korean_saas`
3. **공통 의도** (Mode A의 5축 분류 일부): page_type, language, dials
4. **보존 조건** (Mode B의 자연어 추출): "로고는 그대로", "색상은 우리 브랜드"

### Step 2. _index.json 매칭 알고리즘 실행

`reference-library/_index.json`의 `matching_algorithm` 적용:

#### 2.1 직접 slug 매칭 (가장 강한 시그널)
```
사용자 발화에 정확한 slug 매칭 → 100점
"토스처럼" → toss = 100점
"Linear 풍" → linear = 100점
```

#### 2.2 카테고리 가중치 점수
```
사용자 의도 키워드와 categories[].tags + scenarios 매칭:
- 일치 1개당 +5점
- "한국 핀테크" → korean_saas (5점) + fintech (5점) = 10점
```

#### 2.3 최종 후보 3개 선정
- 직접 매칭 1개 + 카테고리 매칭 상위 2개 = 3개
- 직접 매칭 부재 시 카테고리 상위 3개
- 매칭 0점이면 fallback (Linear, Vercel, Stripe — 글로벌 톱)

### Step 3. 사용자에게 3 후보 제시 + 선택 받기

```markdown
## 🎯 calm-design Mode C — 매칭 결과

요청: "한국 핀테크 차분한 SaaS 로그인 화면"

### 3 후보 (의도 매칭 점수 순)

1. **🥇 토스 (Toss) — 100점**
   - 카테고리: korean_saas + fintech
   - 시그니처: TossBottomSheet, TossInput, TossPrimaryButton
   - 톤: 차분한 한국 핀테크, Pretendard, 단일 Toss Blue 액센트

2. **🥈 Stripe — 15점**
   - 카테고리: fintech (글로벌)
   - 톤: Signature purple gradients, weight-300 elegance
   - 한국어 환경엔 일부 보정 필요

3. **🥉 토스 (Toss) — 한 번 더 (백업)**
   - 카테고리 fallback

### 어떻게 진행하시겠어요?

1. **🥇 토스 채택** — Mode C 풀 진행
2. **🥈 Stripe 채택** — Mode C 풀 진행
3. **혼합** — "토스 컬러 + Stripe 모션" 같은 합성
4. **다른 레퍼런스** — 직접 명시 (예: "Notion으로 해주세요")
5. **Mode A로 폴백** — 레퍼런스 없이 일반 생성
```

### Step 4. 보존 계약 (Mode B와 공통 — 사용자 명시 보존 조건)

사용자가 "로고는 그대로", "Toss Blue는 유지" 명시 시 Mode B Step 4의 자연어 추출 적용. 보존 계약은 Step 5 변환 시 강제.

### Step 5. 레퍼런스 DESIGN.md 로드

선택된 브랜드의 풀 DESIGN.md 로드:

```python
# 한국 시범 3개는 자체 작성 본문 보유
load: reference-library/korean/toss/DESIGN.md  → 즉시 사용 가능

# 글로벌 67개는 메타만 보유
if reference-library/global/{brand}/DESIGN.md 부재:
    print("⚠️ 풀 DESIGN.md 부재. 다음 명령으로 다운로드 후 재시도:")
    print("    npx getdesign@latest add {brand}")
    print("또는 메타데이터(_index.json)만으로 진행 (요약 톤만 적용)")
    fallback to metadata-only mode
```

### Step 6. 사용자 의도와 레퍼런스 합성

레퍼런스 DESIGN.md의 9-섹션을 **베이스**로, 사용자 의도(Mode A 5축 분류)를 **변형**으로 적용:

- **베이스 (레퍼런스에서)**: Visual Theme·Color Palette·Typography·Component Stylings·시그니처 패턴
- **변형 (사용자 의도에서)**: Page Type별 섹션 구조, 사용자 명시 콘텐츠, 보존 조건
- **calm-design 표준 강제**: 30 Pre-Flight, 50 Anti-Slop, LANGUAGE=ko면 Pretendard

#### 합성 예시 (토스 + 인증 폼 페이지)

```yaml
base_from_reference (toss):
  colors: { canvas: white, ink: #191F28, accent: #3182F6 }
  typography: Pretendard
  signature_components: [TossInput, TossPrimaryButton]

user_intent_overlay:
  page_type: auth (로그인 폼)
  preserved_logo: true
  custom_copy: "다시 만나요!"

result:
  → Color from toss (변경 X)
  → Typography from toss (Pretendard 강제)
  → Components from toss (TossInput·TossPrimaryButton 사용)
  → Layout from auth page type (단일 카드 중앙)
  → Copy from user (preserved_logo 유지)
```

### Step 7. 9-섹션 DESIGN.md 생성 (합성 결과)

`references/design-md-spec.md` 표준 따라 새 DESIGN.md 작성. 헤더에 매칭 메타:

```markdown
> Generated by calm-design · LANGUAGE: ko · VARIANCE: 4 · MOTION: 4 · DENSITY: 3 · Mode: C
> Inspired by: 토스 (toss.im) — calm-design reference-library/korean/toss
> Match score: 100 (direct slug match)
> Preserved from user: ["logo", "custom_copy"]
```

### Step 8. 출력 변환 + 셀프-크리틱 (Mode A·B와 공통)

선택한 출력 엔진(html-tailwind / react-shadcn / json-render-spec) 적용 + 30 Pre-Flight + 셀프-크리틱 자동 트리거.

### Step 9. 산출물 5가지 사용자 전달

```markdown
## ✅ calm-design Mode C 완료

[ DESIGN.md ]                  Inspired by 토스 — 합성된 9-섹션
[ 코드 출력 ]                  TossInput·TossPrimaryButton 사용한 인증 폼
[ Pre-Flight Report ]          30/30 ✅ + 한국어 Pretendard 강제 통과
[ Match Report ]               매칭 점수 + 3 후보 + 사용자 선택
[ 다이얼+모드 한 줄 ]          LANGUAGE=ko · VARIANCE=4 · MOTION=4 · DENSITY=3 · Mode C · Inspired by toss
```

## 매칭 알고리즘 의사 코드

```python
def match_reference(user_input: str, intent: dict) -> list[dict]:
    """사용자 입력 → 매칭 후보 3개 반환."""
    candidates = {}
    text = user_input.lower()

    # 1. 직접 slug 매칭 (브랜드명 + "처럼"/"스타일"/"풍")
    for brand in all_brands:
        for trigger in [f"{brand.name}처럼", f"{brand.name} 스타일",
                        f"{brand.name} 풍", f"{brand.name}-style"]:
            if trigger.lower() in text:
                candidates[brand.slug] = 100
                break

    # 2. 카테고리 시그널 매칭
    for cat_id, cat in categories.items():
        score = 0
        for tag in cat.tags + cat.scenarios:
            if tag.lower() in text:
                score += 5
        if score > 0:
            for brand in brands_in_category(cat_id):
                candidates[brand.slug] = candidates.get(brand.slug, 0) + score

    # 3. LANGUAGE=ko면 한국 SaaS에 +10 보너스
    if intent.get("language") == "ko":
        for brand in brands_in_category("korean_saas"):
            candidates[brand.slug] = candidates.get(brand.slug, 0) + 10

    # 4. 상위 3개 선정 (점수 순)
    top_3 = sorted(candidates.items(), key=lambda x: -x[1])[:3]

    # 5. fallback (매칭 0개)
    if not top_3:
        return [
            {"slug": "linear", "score": 0, "fallback": True},
            {"slug": "vercel", "score": 0, "fallback": True},
            {"slug": "stripe", "score": 0, "fallback": True},
        ]

    return [{"slug": s, "score": sc} for s, sc in top_3]
```

이 알고리즘은 `scripts/match-reference.py`(Phase 3.5)로 구현 가능. 현재는 SKILL.md 호출자(LLM)가 _index.json을 직접 읽고 적용.

## Mode C 사용 예시

### 예시 1: 명시 브랜드

```
사용자: "토스처럼 인증 폼 만들어줘. 로고는 우리 거 유지."

→ Step 1: brand=toss (100점), page_type=auth, preserved=[logo]
→ Step 2: 매칭 — toss 100, 다른 후보 부족 → toss 단독
→ Step 3: 사용자 확인 (1 후보만 — 자동 진행 가능)
→ Step 4: 보존 계약 — 로고 유지
→ Step 5: korean/toss/DESIGN.md 로드
→ Step 6: 합성 — 토스 컬러·타이포·시그니처 + 인증 폼 구조 + 사용자 로고
→ Step 7: 새 DESIGN.md 생성
→ Step 8: HTML 출력 + Pre-Flight 통과
→ Step 9: 5 산출물 전달
```

### 예시 2: 카테고리 시그널

```
사용자: "한국 SaaS 차분한 톤 대시보드"

→ Step 1: brand=없음, category=[korean_saas, saas_productivity], page=dashboard
→ Step 2: 점수 — toss 25 (korean+ko language bonus), notion 10, linear 10
→ Step 3: 사용자에게 3 후보 제시 → 선택
→ Step 4-9: 선택된 브랜드로 진행
```

### 예시 3: fallback (매칭 0)

```
사용자: "그냥 멋진 디자인 만들어줘"

→ Step 1: 시그널 부족
→ Step 2: 매칭 0
→ Step 3: fallback — Linear, Vercel, Stripe 글로벌 톱 3 제시
→ 사용자 선택 또는 Mode A로 폴백
```

## 다른 모드와의 분기

Mode C 진행 중 다음 시그널 발견 시 자동 전환:
- "3가지 안" → Mode D (각 베리언트가 다른 레퍼런스 매칭)
- "이거 다듬어줘" + 기존 코드 → Mode B (Upgrade에 레퍼런스 적용)
- "레퍼런스 무시하고 새로 그려" → Mode A 폴백

## 보존 정책 (라이선스 안전)

Mode C 출력은 다음 정책 자동 적용:
- ✅ DESIGN.md 헤더에 "Inspired by {브랜드}" 명시
- ✅ 직접 복제 X — 토스 컬러 hex가 정확히 일치할 수 있으나 brand 시그니처(로고·정확한 typography 라이선스 폰트) 사용 X
- ✅ 사용자 프로젝트의 자체 브랜드 자산 우선 (보존 계약)
- ❌ 브랜드 로고·로고 마크 직접 사용 금지
- ❌ 독점 폰트 (브랜드 전용 폰트) 사용 금지 — system 폰트 또는 Pretendard로

>
> **추가 영향**: [google-labs-code/stitch-skills](https://github.com/google-labs-code/stitch-skills)의 design-md 스킬 (Apache 2.0)

calm-design 스킬이 생성하는 모든 DESIGN.md는 아래 9-섹션 포맷을 정확히 준수한다.

## 산출 위치

생성된 DESIGN.md는 **`.calm-design/DESIGN.md`**로 저장한다. 사용자 프로젝트 루트에 `.calm-design/` 폴더가 없으면 자동 생성.

## 9-섹션 템플릿 (이대로 채워라)

```markdown
# Design System: [프로젝트 이름]

> Generated by calm-design · LANGUAGE: ko · VARIANCE: N · MOTION: N · DENSITY: N · Mode: [A/B/C/D]
> [Match-Reference 모드인 경우] Inspired by: [브랜드명]

## 1. Visual Theme & Atmosphere

(분위기·밀도·디자인 철학을 환기시키는 산문. 토큰 나열이 아니라 "어떤 느낌"인지를 먼저 그린다. 2-4문장.)

예시: "차분한 한국 SaaS 톤. 흰 캔버스에 단일 액센트(Emerald), 충분한 호흡감. 데이터를 강조하되 화려하지 않은, 신뢰감 있는 인터페이스."

## 2. Color Palette & Roles

| 이름 | Hex | 역할 |
|---|---|---|
| Canvas | `#FAFAFA` | 페이지 배경 |
| Surface | `#FFFFFF` | 카드·시트 배경 |
| Ink | `#0A0A0A` | 본문·헤드라인 (Pure Black 절대 금지, Off-Black 사용) |
| Mute | `#71717A` | 보조 텍스트, 메타데이터 |
| Border | `#E4E4E7` | 1px 분할선 |
| Accent | `#10B981` | 단일 액센트 (CTA·강조). 채도 <80% |
| Danger | `#EF4444` | 오류·삭제 |
| Success | `#22C55E` | 성공·완료 |

**필수 규칙**: 한 페이지 액센트 1개만, "LILA BAN"(보라/네온 자주색 그래디언트) 절대 금지, Pure Black `#000000` 절대 금지.

## 3. Typography Rules

**한국어 (LANGUAGE=ko 시 강제)**:
- Family: `Pretendard, system-ui, sans-serif`
- 디스플레이/H1: `text-4xl md:text-5xl lg:text-6xl`, `tracking-tight`, `leading-tight`, `word-break: keep-all`
- 본문: `text-base md:text-lg`, `leading-relaxed`, `max-w-[65ch]` (한국어 65자 ≒ 영문 95자)
- 금지: Inter, Noto Sans KR, Malgun Gothic, Roboto, Apple SD Gothic Neo (시스템 의존)

**영문 (LANGUAGE=en)**:
- Family: `Geist`, `Cabinet Grotesk`, `Outfit`, `Satoshi` 중 하나 강제
- 디스플레이: `tracking-tighter`, `leading-none`
- 금지: Inter, Helvetica (보편화돼 변별력 없음), Times/Georgia (대시보드/SaaS 부적합)

자세한 한국어 타이포 규칙은 `references/typography-ko.md` 참조.

## 4. Component Stylings

각 컴포넌트마다 **Default / Hover / Focus / Active / Disabled / Loading** 상태를 명시. 하나라도 누락 금지.

**Button (Primary)**
- Default: `bg-{accent}` `text-white` `rounded-full` `px-6 py-3` `font-medium`
- Hover: `scale-[1.02]` 300ms `cubic-bezier(0.16, 1, 0.3, 1)` (스프링 피직스)
- Focus: `ring-2 ring-{accent} ring-offset-2`
- Active: `scale-[0.98]`
- Disabled: `opacity-50 cursor-not-allowed`
- Loading: spinner 인라인 + 텍스트 유지 ("Loading..." 같은 공허한 텍스트 X)

**Card (Default)**
- `bg-white` `border border-zinc-200` `rounded-2xl` `p-6`
- Hover (interactive only): `border-zinc-300` 추가, 그림자 X (계층 필요할 때만)
- Empty State: `text-center text-mute` + 컴포지션된 일러스트(SVG), "데이터 없음" 같은 빈 텍스트만 X

**Input**
- 라벨은 위, 에러는 아래, 스페이싱 `gap-2`
- Border: `border-zinc-200`, Focus: `border-{accent}` + `ring-2 ring-{accent}/20`

## 5. Layout Principles

- **Macro 화이트스페이스**: 섹션 패딩 최소 `py-24 md:py-32 lg:py-40`. 디자인이 호흡하게 하라.
- **Max-width**: `max-w-7xl mx-auto px-4 md:px-6` 강제 (1400px 이상 풀-블리드 X, 가독성 깨짐)
- **그리드**: CSS Grid 우선, Flexbox는 1차원만. `grid-cols-12` 기반.
- **Variance ≥ 5**: 인접 섹션은 **다른 레이아웃** 사용. 같은 패턴 반복 시 Pre-Flight 실패.
  - Hero(Split) → Features(Bento) → Testimonials(Masonry) → CTA(Full-bleed) → Footer(Minimal)
- **금지**: `h-screen` (iOS 100vh 버그) → `min-h-[100dvh]` 강제. 3-column equal cards → Bento/Zig-Zag/Masonry.
- **모바일**: 다중 컬럼은 `<768px`에서 단일 컬럼 강제. Horizontal scroll 금지.

## 6. Depth & Elevation

- **Shadow는 계층이 필요할 때만**. 무지성으로 모든 카드에 `shadow-lg` 금지.
- **Tinted shadow**: 어두운 배경이면 `shadow-[0_2px_8px_rgba(0,0,0,0.4)]`, 컬러 배경이면 그 색의 어두운 톤으로 틴팅.
- **Inset highlight** (Double-Bezel 카드용): `shadow-[inset_0_1px_1px_rgba(255,255,255,0.15)]`
- **z-index 계층**: nav=40, overlay=50, noise=60. 절대 그 이상 사용 X.
- **금지**: 네온/외부 글로우 (`shadow-[0_0_30px_#7c3aed]` 같은 자주색 글로우 절대 금지 — LILA BAN)

## 7. Motion & Interaction

- **Spring Physics 기본**: `stiffness: 100, damping: 20` (프리미엄·무거운 느낌). Linear easing X.
- **GPU 친화 속성만**: `transform`, `opacity`. `top/left/width/height` 애니메이션 절대 X.
- **Staggered Entry**: 리스트 즉시 마운트 X. `delay: index * 50ms` cascade.
- **Perpetual Micro-Interactions** (MOTION ≥ 6일 때): 활성 컴포넌트에 무한 루프 (Pulse, Float, Shimmer).
- **React 모드**: Framer Motion `useMotionValue` / `useTransform` 강제. `useState` 기반 애니메이션 X.
- **HTML 모드**: Motion One(4KB) 사용. GSAP는 명시 요청 시만.

## 8. Responsive Behavior

- **Breakpoints**: 모바일 `<768px`, 태블릿 `768–1024px`, 데스크톱 `≥1024px`
- **Touch targets**: 최소 44×44px (iOS HIG 준수)
- **Typography scaling**: `clamp(1rem, 2vw + 0.5rem, 1.25rem)` 형태로 부드럽게
- **Image behavior**: Inline-typography 이미지(헤드라인 단어 사이 삽입)는 모바일에서 헤드라인 아래로 떨어뜨리기
- **Navigation**: 데스크톱 수평 → 모바일 햄버거 또는 바텀 시트

## 9. Anti-Patterns (Project-Specific Banned)

**역할 분리 (중요)**:
- **글로벌 안티-슬롭**(모든 calm-design 출력에 자동 적용되는 50+ 항목 — Inter 금지, Pure Black 금지, LILA BAN, 3-column equal cards 금지 등)은 `references/ai-tells-blocklist.md`가 전담한다. **이 섹션에 중복 기재하지 않는다.**
- 이 섹션은 **이 프로젝트에만 해당하는 금지 항목**(브랜드 톤·도메인 제약·법적 요구사항·이전 사용자 피드백)만 담는다.
- 따라서 이 섹션은 **비어 있을 수 있다.** 비어있으면 다음 한 줄로 명시: `Project-specific bans: None (글로벌 안티-슬롭만 적용됨)`

**프로젝트 고유 금지 작성 카테고리**:
- **브랜드 톤**: "유머러스 카피 금지" (B2B 금융), "광고적 표현 금지" (공공기관·의료)
- **도메인 제약**: "경쟁사 시그니처 색상 금지", "특정 컴포넌트 패턴 금지" (이전 시도 실패)
- **법적·접근성**: "WCAG 2.2 AA 미달 색상 대비 금지", "특정 단어 마케팅 금지" (금융·의료 규제)
- **이전 피드백**: "사용자 테스트에서 거부된 패턴 제외"

**작성 방식 (모드별)**:
- **Generate 모드**: 사용자 입력에서 "X는 빼줘", "Y는 우리 톤이 아니야" 등의 시그널 자동 추출 후 기재
- **Match-Reference 모드**: 레퍼런스 브랜드의 anti-patterns 섹션 차용
- **Multi-Variant 모드**: 각 베리언트마다 서로 다른 anti-patterns 가능

## 검증 기준 (스킬이 자체적으로 체크)

DESIGN.md를 출력하기 전, 스킬은 다음을 자동 검증:

- [ ] 9개 섹션 모두 비어있지 않음 (각 섹션 최소 50자 이상)
- [ ] 색상 hex가 모두 유효 (`#` + 6자리 hex)
- [ ] Pure Black `#000000` 미포함
- [ ] Inter 폰트 미언급 (LANGUAGE=ko 시 Pretendard 명시)
- [ ] Section 4의 모든 컴포넌트가 6개 상태(Default/Hover/Focus/Active/Disabled/Loading) 명시
- [ ] Section 5에 `min-h-[100dvh]` 명시 (full-height 섹션 있을 때)
- [ ] Section 9가 비어있지 않음 (최소 3개 anti-pattern)

하나라도 실패하면 DESIGN.md 재생성 (최대 3회).

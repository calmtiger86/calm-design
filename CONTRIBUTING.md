calm-design 프로젝트에 기여해 주셔서 감사합니다. 이 문서는 기여하는 방법을 안내합니다.

## 📝 기여 절차

### 1. 이슈 먼저 열기

큰 변경(새 모드·새 출력 엔진·새 카테고리)은 PR 전에 **반드시 이슈를 열어** 논의:

- 변경 동기 (어떤 문제 해결?)
- 영향 범위 (어떤 파일 수정?)
- 호환성 (기존 사용자에게 깨지는 변경?)

작은 변경(매핑 1개 추가, 안티-패턴 1개 추가, 오타 수정)은 바로 PR 가능.

### 2. Fork → Branch

```bash
git clone https://github.com/{your-username}/calm-design
cd calm-design
git checkout -b feature/내가-할-변경
```

브랜치 이름:
- `feature/...` — 새 기능
- `fix/...` — 버그 수정
- `docs/...` — 문서만
- `reference/...` — reference-library 추가

### 3. 변경 후 검증

PR 전에 반드시 확인:

- [ ] **메인 SKILL.md 6KB 한도 초과 안 함** (`wc -c SKILL.md` ≤ 6144)
- [ ] 새로 추가한 한국어 텍스트에 `word-break: keep-all` 적용
- [ ] 새로 추가한 코드에 Inter 폰트 미사용
- [ ] 새로 추가한 코드에 Pure Black `#000000` 미사용
- [ ] 예시 추가 시 calm-design 자체로 생성 가능한 결과여야 함 (자기 적용 통과)
- [ ] 마크다운 링크가 모두 유효
- [ ] 한국어 README 업데이트 시 영문 README도 함께 업데이트

### 4. PR 작성

PR 제목: `[영역] 한 줄 요약`
- `[a11y] Pre-Flight 22번 항목 추가: WCAG 2.2 AA 색상 대비`
- `[ko-mapping] "장바구니 아이콘" → "Cart Icon Button" 매핑 추가`
- `[reference] 라인(LINE) DESIGN.md 추가`

PR 본문:
- 무엇을 바꿨는가
- 왜 바꿨는가 (이슈 번호 링크)
- 어떻게 검증했는가
- 깨지는 변경(breaking change) 여부

## 🎨 reference-library/ 기여 정책 (중요)

레퍼런스 브랜드 추가는 다음 정책을 **반드시** 준수:

### 허용

- ✅ 공개 디자인 시스템 문서·블로그 포스트·인터뷰 기반 분석
- ✅ "Inspired by" 명시 (직접 복제 아닌 영감 표시)
- ✅ 추상적 디자인 인상 ("차분한", "데이터 중심", "단일 액센트")
- ✅ 일반적 폰트 패밀리 정보 (Pretendard, Inter 같은 OSS 폰트만)

### 금지

- ❌ 브랜드 로고 직접 첨부
- ❌ 실제 브랜드 컬러 hex 값 정확 복제 (대신 "Toss Blue 톤의 파란색" 같은 추상화)
- ❌ 독점 폰트 (커스텀 폰트, 라이선스 필요한 폰트) 정보
- ❌ 비공개 디자인 시스템 문서 인용
- ❌ 경쟁사를 공격하는 표현

### 양식 (`reference-library/{brand}/DESIGN.md`)

```markdown
# Inspired by: {Brand Name}

> Generated by calm-design · LANGUAGE: {ko|en} · VARIANCE: N · MOTION: N · DENSITY: N
> Inspired by: {Brand Name} ({domain.com}) — {1줄 인상}

## 1. Visual Theme & Atmosphere
(공개 자료 기반 분석)

## 2. Color Palette & Roles
(추상화된 색상 톤 — 정확한 hex 복제 X)
...
(나머지 9-섹션 표준)
```

## 🌐 한국어·영어 양립 정책

calm-design은 한국어 1순위지만 영어권 사용자도 사용 가능:

- README는 한국어 우선, README-EN.md 별도
- 코드 주석은 한국어 OK, 단 함수명·변수명은 영어
- 안티-패턴 카피 차단 목록은 한·영 양립 (한국어 카피 클리셰도 추가 환영)
- LANGUAGE=auto 모드의 정확도 개선 PR 환영

## 💡 좋은 기여의 예

### 예 1: 새 매핑 추가

```diff
# references/prompt-enhancement.md
| "햄버거 버튼" | Hamburger Menu | mobile menu, nav drawer |
+ | "FAB", "플로팅 버튼" | Floating Action Button | fab, primary floating cta |
```

### 예 2: 새 안티-패턴 추가

```diff
# references/ai-tells-blocklist.md (Phase 1+ 항목)
+ ### 31. ❌ 한국어 페이지에 영문 lorem ipsum 사용 금지
+ - ✅ 대신: 의미 있는 한국어 더미 텍스트 또는 실제 콘텐츠 초안
+ - 💡 이유: 한국어 페이지에 라틴 lorem은 즉시 placeholder 인상 + 줄바꿈 정책 무관
```

### 예 3: 작은 버그 수정

```diff
# output-engines/html-tailwind.md
- <input class="w-full h-10 px-3 rounded-md border border-zinc-200 focus:outline-none focus:ring-2 focus:ring-accent" placeholder="you@example.com" />
+ <input type="email" class="w-full h-10 px-3 rounded-md border border-zinc-200 focus:outline-none focus:ring-2 focus:ring-accent" placeholder="you@example.com" autocomplete="email" />
```
